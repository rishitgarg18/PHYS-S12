### Week 9: Networking and IoT As my Final Project involves working with a gyroscope and an ESP-32, I used this week's assignment to create a web server which displays my readings of a gyroscope. The gyroscope is powered by an ESP-32. To do so, I followed the "Random Nerd Tutorials" website for such a program and I will link the website down at the bottom of this webpage. To complete this assignment, I needed an html file and a css file for the webpage, a java script file and finally an arduino code to run the ESP-32. All these files can be found in the "RandomNerd Tutorial" linked at the bottom. TTo create such as sensor, one has to be careful about whether or not they have the required drivers already installed. You will need the following drivers for this sensor: Adafruit MPU 6050, Adafruit Unified Sensor, Adafruit Bus IO, Async Web Server Libraries, Arduino_JSON, and lastly the ESP 32 File System Uploader on Arduino IDE. Using the File System Uploader on Arduino IDE can be tricky and I would recommend using this link to get around with it: <br> <a href="https://randomnerdtutorials.com/install-esp32-filesystem-uploader-arduino-ide/">Install ESP-32 File System Uploader</a> One issue that I face while trying to run the program is that I would get an error such as "not in download mode" or "board not found". To solve this error, I would recommend holding down the "Boot" button onyour ESP-32 wwhen the sketch is complied and you see the message "Connecting...." Lastly, be sure that ypur computer and the ESP-32 are connected to the same Wifi for the program to run. }); server.serveStatic("/", SPIFFS, "/"); server.on("/reset", HTTP_GET, [](AsyncWebServerRequest *request){ gyroX=0; gyroY=0; gyroZ=0; request->send(200, "text/plain", "OK"); }); server.on("/resetX", HTTP_GET, [](AsyncWebServerRequest *request){ gyroX=0;
