### Week 6: Electronic Input Devices For this week's assignment, I worked on three kinds of sensors: a <a href="https://create.arduino.cc/projecthub/Marcazzan_M/how-easy-is-it-to-use-a-thermistor-e39321">Thermistor</a>, a <a href="https://randomnerdtutorials.com/arduino-mpu-6050-accelerometer-gyroscope/">Gyroscope</a>, a <a href="https://nathanmelenbrink.github.io/intro-dig-fab/06_input/index.html">Flex Sensor</a> <h2> 1. Thermistor </h2> The Thermistor required the following materials: a 10k ohm resistor, a Thermistor, Arduino UNO/Metro, Jumper Wires, and a Breadboard. Here is the Schematic Representation: ![schematic](termistore_bb.jpg) And here is the code for the thermistor: <pre><code class="language-arduino"> //Thermometer with thermistor /*thermistor parameters: * RT0: 10 000 Ω * B: 3977 K +- 0.75% * T0: 25 C * +- 5% */ //These values are in the datasheet #define RT0 10000 // Ω #define B 3977 // K //-------------------------------------- #define VCC 5 //Supply voltage #define R 10000 //R=10KΩ //Variables float RT, VR, ln, TX, T0, VRT; void setup() { Serial.begin(9600); T0 = 25 + 273.15; //Temperature T0 from datasheet, conversion from Celsius to kelvin } void loop() { VRT = analogRead(A0); //Acquisition analog value of VRT VRT = (5.00 / 1023.00) * VRT; //Conversion to voltage VR = VCC - VRT; RT = VRT / (VR / R); //Resistance of RT ln = log(RT / RT0); TX = (1 / ((ln / B) + (1 / T0))); //Temperature from thermistor TX = TX - 273.15; //Conversion to Celsius Serial.print("Temperature:"); Serial.print("\t"); Serial.print(TX); Serial.print("C\t\t"); Serial.print(TX + 273.15); //Conversion to Kelvin Serial.print("K\t\t"); Serial.print((TX * 1.8) + 32); //Conversion to Fahrenheit Serial.println("F"); delay(500); } </code></pre> Here are a few photos from my working of the Thermistor: <br> ![thermistor 1](IMG_6052.jpg) ![thermistor 2](IMG_6053.jpg) <video controls> <source src="IMG_6055.MOV.mp4" type="video/mp4"> </video> <h2> 2. The Flex Sensor </h2> I prepared a Flex Sensor using two copper plates, some foam, an Arduino UNO, and a couple of Jumper Wires. The Flex Sensor was prepared using the Tx-Rx method of parallel plate capacitors. Here is the code that I used for the Flex Sensor: <pre><code class="language-arduino"> int analog_pin = A3; int tx_pin = 4; int read_high; int read_low; int diff; void setup() { pinMode(tx_pin, OUTPUT); // Pin 4 provides the voltage step Serial.begin(9600); } void loop() { digitalWrite(tx_pin,HIGH); // Step the voltage high on conductor 1. read_high = analogRead(analog_pin); // Measure response of conductor 2. delayMicroseconds(100); // Delay to reach steady state. digitalWrite(tx_pin,LOW); // Step the voltage to zero on conductor 1. read_low = analogRead(analog_pin); // Measure response of conductor 2. diff = read_high - read_low; // desired answer is the difference between high and low. Serial.println(diff); } </code></pre> Here is a photo of the Flex Sensor: ![Flex Sensor](IMG_6063.jpg) Also, Here is a table that shows the various readings I obtained using the Flex Sensor. I used whatever objects I could find and then weighed them on a weighing scale to know their mass in grams. I then put these objects on the Flex sensor to obtain various readings. Here is a summary of those readings: ![Table](Screenshot8.png) One interesting thing to note is that even though the Stationery kit is 50 grams heavier than my phone, there was quite a difference between the outputs of the two objects. This is probably because the kit has a larger surface area and is hence covering more area of the Flex Sensor. Also, the ouputs of all these objects do not have units and do not linearly match the mass or any other quantity of the object. The outputs are just relative measurements that tell us how much the foam is being compressed. Finally, the Flex Sensor readings did not start from zero.
